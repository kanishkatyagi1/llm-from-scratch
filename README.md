# 🧠 LLM From Scratch

A ground-up implementation of a GPT-style Large Language Model in PyTorch — from a character-level Bigram baseline to a full multi-head Transformer with self-attention, trained on the OpenWebText dataset.

Built to deeply understand how modern language models work, not just use them.

---

## 📌 Project Overview

This project walks through the full lifecycle of building and training a language model:

1. **Bigram Model** — a simple character-level baseline to establish the pipeline
2. **GPT Transformer** — multi-head self-attention, positional embeddings, feed-forward layers, residual connections, and layer normalization
3. **Data Pipeline** — streaming the OpenWebText corpus (~40GB) via HuggingFace Datasets without full download
4. **Training Script** — configurable CLI trainer with checkpoint saving
5. **Chatbot Inference** — interactive prompt-completion interface powered by the trained model

## 🏗️ Model Architecture

```
GPTLanguageModel
├── Token Embedding          (vocab_size → n_embd)
├── Positional Embedding     (block_size → n_embd)
├── Transformer Blocks × 3
│   ├── Multi-Head Attention (4 heads, causal masking)
│   ├── Feed-Forward Network (n_embd → 4×n_embd → n_embd)
│   └── Layer Norm + Residual Connections
├── Final Layer Norm
└── Linear Head              (n_embd → vocab_size)
```

| Hyperparameter | Value |
|---|---|
| Embedding dim (`n_embd`) | 384 |
| Attention heads (`n_head`) | 4 |
| Transformer layers (`n_layer`) | 3 |
| Block / context size | 64 tokens |
| Dropout | 0.1 |
| Learning rate | 3e-4 |
| Optimizer | AdamW |

## 🚀 Getting Started

### 1. Clone the repo

```bash
git clone https://github.com/YOUR_USERNAME/llm-from-scratch.git
cd llm-from-scratch
```

### 2. Install dependencies

```bash
pip install torch datasets tqdm
```

### 3. Prepare the data

Streams 10,000 samples from OpenWebText (no full download needed):

```bash
python data_extract.py
```

This generates `openwebtext/output_train.txt`, `output_val.txt`, and `vocab.txt`.

### 4. Train the model

```bash
python training.py -batch_size 32
```

Trains for 200 iterations, prints loss every 50 steps, and saves checkpoint to `model-01.pkl`.

### 5. Run the chatbot

```bash
python chatbot.py -batch_size 32
```

Then type a prompt and the model will complete it:

```
Prompt:
Once upon a time

Completion:
Once upon a time the world was a very different place...
```

## 🔧 Extending the Project

Some natural next steps if you want to experiment:

- Increase `max_iters` and `sample_rate` in `data_extract.py` for better quality output
- Add a learning rate scheduler (cosine decay works well)
- Scale up `n_layer`, `n_head`, and `n_embd` if you have a more powerful GPU
- Replace character-level tokenization with BPE (e.g. `tiktoken`)
- Add gradient clipping (`torch.nn.utils.clip_grad_norm_`)

## 💡 Key Concepts Implemented

- **Causal self-attention** with masking (tokens can only attend to past tokens)
- **Multi-head attention** for learning different relationship patterns in parallel
- **Positional embeddings** to give the model awareness of token order
- **Residual connections** and **Layer Normalization** for training stability
- **Memory-mapped file I/O** (`mmap`) for efficient large-file batching
- **Streaming dataset** loading via HuggingFace to avoid downloading 40GB upfront
